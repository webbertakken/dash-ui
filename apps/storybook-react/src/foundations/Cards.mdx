# Cards

`Card` is the workhorse of every reference page. It supplies the tile background, the rounded
corner, the interior padding, and — critically — the `<h3>` title rhythm that everything else
(charts, stats, tables, lists) sits below. Reproduce the card contract verbatim and userland tiles
read the same as the Dashboard story.

If you only learn one rule from this page: **render `<Card>` as a direct child of `.grid`, never
wrap it.** Extra wrappers are the single biggest cause of card padding / radius / title drift.

## The contract

```tsx
import { Card, Pill } from '@w5-ui/react'

;<Card span={4}>
  <h3>
    Internet <Pill variant="success">Online</Pill>
  </h3>
  {/* chart, list, table — whatever the tile shows */}
</Card>
```

That is the whole thing. The `<h3>` becomes the tile title automatically because the `.card h3` CSS
rule sets its size (12px), weight (500), colour (`--text-muted`), and flex layout (title on the
left, status / "More" link pushed to the right). Don't restyle the heading. Don't replace it with an
`<h2>` or a custom `<div className="title">` — those skip the rule and the tile reads broken.

## Card vs Stat

A `<Stat>` is a `<Card>` with a pre-wired hero number, unit, sub-line, and delta. Reach for it any
time the tile's primary content is one number you want the eye to land on.

```tsx
import { Stat } from '@w5-ui/react'

;<Stat
  span={4}
  label="Internet"
  value="847"
  unit="Mbps · down"
  sub="18 ms · 0.0% loss"
  delta="+12% 24h"
  deltaDir="up"
/>
```

| Reach for…   | When the tile is…                                                           |
| ------------ | --------------------------------------------------------------------------- |
| `<Stat>`     | One headline number (with optional unit, sub-line, delta)                   |
| `<Card>`     | Anything else — chart, table, list, mixed content                           |
| `<CardTitle>`| An `<h3>` rendered in markup where the raw element is awkward (codegen, MDX) |

`<Stat>` already renders its own `<Card>`, so **never nest `<Stat>` inside `<Card>`**. Render it at
the top level of `.grid` alongside other cards.

## Sizing with `span`

`span` controls the tile's column width on the 12-column `.grid`. Pick a value that divides 12
cleanly:

| Tile width | `span` | Typical use                          |
| ---------- | ------ | ------------------------------------ |
| Third      | `4`    | `<Stat>`, single donut, small list   |
| Half       | `6`    | Two-column comparison, mid-size chart|
| Two-thirds | `8`    | Wide table, line / area chart        |
| Full       | `12`   | Full-bleed time series, sankey       |

4+4+4, 8+4, 6+6, and 12 all snap to one row. **Avoid `span={5}` and `span={7}`** — they leave dead
pixels and force the next row to wrap unevenly. If `span` is omitted, the card spans the row.

## Anatomy

| Layer            | Where the values live                                                              |
| ---------------- | ---------------------------------------------------------------------------------- |
| `.card` shell    | 16px (`--space-4`) padding, `--radius-md` (6px) corner, `--surface-1` background   |
| `.card h3` title | 12px / weight 500 / `--text-muted`, flex row with 8px (`--space-2`) inline gap     |
| Title trailing   | Anything after the title text in the `<h3>` (Pill, More link) is pushed right     |
| Body             | Any element after the `<h3>`. Charts auto-fill, lists / tables stack on the 4px grid |

## Content patterns

### Headline number → use `<Stat>`

```tsx
<Stat span={4} label="Clients" value="124" unit="online" delta="+3" deltaDir="up" />
```

### Headline number + chart → `<Submeta>` for the supporting line

When the tile wants a hero number _and_ a chart underneath, `<Stat>` can't help (it owns the whole
card body). Compose the parts by hand and use `<Submeta>` for the muted line that sits between the
number and the chart — it matches the `.submeta` rhythm without an inline class string.

```tsx
import { Card, Pill, StatNumber, Submeta, Delta, Unit, Sparkline } from '@w5-ui/react'

;<Card span={4}>
  <h3>
    Internet <Pill variant="success">Online</Pill>
  </h3>
  <StatNumber>
    847<Unit>Mbps · down</Unit>
  </StatNumber>
  <Submeta>
    ↑ 312 Mbps · 18 ms · 0.0% loss · <Delta dir="up">+12% 24h</Delta>
  </Submeta>
  <Sparkline />
</Card>
```

`<Delta>` is the typed wrapper for the colour-coded trend span. Pass `dir="up" | "down" | "flat"` —
green / red / neutral colours come from the tokens, and a visually-hidden "Trend: " prefix is
emitted so screen readers announce the direction independent of any arrow glyph in the copy.

`<Unit>` is the typed wrapper for the small trailing label that sits next to the hero number (or in
a card title row): renders `<span className="unit">` so the canonical muted colour and 12 px size
come from `dashboard.css` instead of an inline override. `<Stat>` already wires this internally;
reach for `<Unit>` when you compose `.stat` by hand (number above a chart, or a unit suffix in an
`<h3>` title).

`<StatNumber>` is the typed wrapper for the hero-number row itself: renders `<div className="stat">`
so the canonical 30 px size, semibold weight, tabular figures, and primary text colour come from
`dashboard.css` rather than inline `font-size` / `font-weight` overrides. `<Stat>` (the prefab tile)
wires this internally; reach for `<StatNumber>` when you compose `.stat` by hand inside a `<Card>`
that also hosts a chart underneath.

### Title + chart

```tsx
<Card span={8}>
  <h3>Throughput</h3>
  <LineChart series={series} />
</Card>
```

### Title with status pill

```tsx
<Card span={4}>
  <h3>
    Internet <Pill variant="success">Online</Pill>
  </h3>
  <Sparkline data={uplink} />
</Card>
```

### Title with "More →" affordance

The `.card h3 .more` rule already styles a trailing inline link. Use `<CardMore>` for the typed
wrapper — don't roll your own button in the title row.

```tsx
import { Card, CardMore, RankedList } from '@w5-ui/react'

<Card span={6}>
  <h3>
    Top clients <CardMore href="/clients">All →</CardMore>
  </h3>
  <RankedList items={topClients} />
</Card>
```

`<CardMore>` is a thin typed wrapper over `<a className="more">`. Forwards all anchor props (`href`,
`target`, `onClick`, …) so it doubles as a button-shaped link without losing keyboard semantics.

### Mixed table tile

Tables nested inside a card inherit the card's interior padding. For full-bleed tables that should
sit edge-to-edge in the tile, lift them out of `<Card>` and wrap in `.table-shell` instead — see
**Foundations / Tables**.

## Common mistakes (and the fix)

| Symptom                                       | Likely cause                                                | Fix                                                                                  |
| --------------------------------------------- | ----------------------------------------------------------- | ------------------------------------------------------------------------------------ |
| Tile padding looks wrong                      | Re-implementing `.card` with a custom `<div>`               | Use `<Card>` — radius, padding, and `<h3>` rhythm ship together                      |
| Card has double padding inside                | A wrapper `<div style={{ padding }}>` inside `<Card>`       | Render the content directly inside `<Card>`; the card already pads 16px              |
| Title font / colour doesn't match dashboard   | Custom `<h2>` / `<div>` instead of `<h3>`                   | Use `<h3>` inside `<Card>` — `.card h3` sets size, weight, colour                    |
| Status pill in title is left-aligned          | Wrapped in an extra `<span>` or `<div>` after the title text| Place the `<Pill>` as a direct child of the `<h3>`; the flex rule pushes it right    |
| Two cards in a row don't line up vertically   | Different `span` values, or one wrapped in an extra `<div>` | Match `span` values. No wrappers between `.grid` and `<Card>`                        |
| Stat looks cramped or doubly-padded           | `<Stat>` nested inside another `<Card>`                     | `<Stat>` is a card already — render at the top level of `.grid`                      |
| Card corner doesn't match modal corner        | Hand-typed `border-radius: 8px`                             | Cards use `--radius-md` (6px). Modals/drawers use `--radius-lg` (12px). Don't mix    |
| Card content bleeds past the rounded corner   | Inner element with `border-radius: 0` and a coloured bg     | Add `overflow: hidden` only when you really need bleed; otherwise pad the inner edge |
| Tile widths drift between motifs              | Card background hardcoded to a hex                          | Use `--surface-1` (or let `<Card>` set it). Tokens flip with `data-motif`            |

## CSS hooks userland can rely on

Stable class names shipped by `@w5-ui/tokens/dashboard.css` (re-exported via `@w5-ui/react/styles.css`
and `@w5-ui/svelte/styles.css`):

- `.card` — the tile shell. Use through `<Card>` / `<Stat>`
- `.card h3` — the title rhythm (size, weight, colour, flex). Use a plain `<h3>` (or `<CardTitle>` when the raw element is awkward — codegen, MDX, mixed JSX trees) and inherit it
- `.card h3 .more` — the trailing "More →" affordance inside a title. Use through `<CardMore>`
- `.stat` — the hero-number row (30 px, semibold, tabular). Use through `<Stat>` / `<StatNumber>`
- `.submeta` — the muted supporting line under a `.stat` number. Use through `<Submeta>`
- `.delta-up` / `.delta-down` — colour-coded trend text. Use through `<Delta dir="up | down | flat">`
- `.unit` — small trailing label after a hero number or inside a title row. Use through `<Unit>`

Everything else (raw colours, ad-hoc paddings, custom radii) should come from tokens, not bespoke
values. See **Foundations / Spacing** and **Foundations / Colours**.
