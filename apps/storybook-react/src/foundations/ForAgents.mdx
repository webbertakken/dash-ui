# For agents

A single condensed brief for **agentic implementers** (Claude Code, Cursor, Copilot, etc.) wiring
a userland app on top of `@w5-ui/react` or `@w5-ui/svelte`. The other Foundations pages explain
**why** the contract is shaped the way it is; this page is the **what** in directive form, sized
to drop straight into a userland `AGENTS.md` / `CLAUDE.md` / Cursor rules file.

> If you are a human, skim this once and bookmark **Foundations / Setup** + **Recipes**. The
> wording below is deliberately terse so an LLM can keep the whole contract in its working set.

## Paste-into-userland brief

Copy this block verbatim into `AGENTS.md` (or `CLAUDE.md`, or `.cursor/rules/dash-ui.md`) at the
root of any userland project that consumes `@w5-ui/react` or `@w5-ui/svelte`. It is written as
imperative rules — keep it short so it stays inside the model's prompt budget.

```md
# Dash UI — implementation contract

Userland renders like the reference dashboard **only** when these rules hold. Every visual drift
("spacing feels off", "colours look washed out", "header doesn't stick") maps to a rule below.

## Setup (do once)

- Import `@w5-ui/react/styles.css` (or `@w5-ui/svelte/styles.css`) **once** at the app entry.
  Never import `@w5-ui/tokens/tokens.css` alone — `dashboard.css` is what wires `.app`/`.grid`
  /`.card`. Tailwind users: import `@w5-ui/tokens/tailwind-full.css` instead.
- Set `<html data-motif="dark">` (or `"light"`) **inline in `index.html`**, never inside a
  `useEffect`. Anti-FOUC requires the attribute before the first paint.
- Load Inter (UI) + JetBrains Mono (numeric cells). System fonts shift the type scale ~1px.

## Page shell (every page, no exceptions)

- `<AppLayout pageLabel topbar={<Topbar …/>} sidebar={<Sidebar …/>}>` wraps the whole window.
  Never hand-roll `.app` / `.workspace` / `<main>`.
- Inside `AppLayout`, render at most three siblings: optional `<Banner>`, then `<PageHeader>`,
  then `<div className="grid">…</div>`. No extra wrapper `<div>` around the grid.
- `<PageHeader title actions>` owns the sticky header. Never re-implement `.ph-bar` /
  `.ph-title` / `.ph-actions` by hand.

## Inside the grid

- `.grid` is `repeat(12, 1fr)`, 12px gap, 16px × 24px padding. **Never** override `gap` /
  `padding` on `.grid` or wrap it.
- Every direct child is `<Card span={n}>` or `<Stat span={n}>`. Use `span` in {4, 6, 8, 12} —
  never 5 or 7.
- For a hero number, use `<Stat>` (it _is_ a `<Card>`). For richer tiles, compose inside
  `<Card>` and let `.card h3` style the title — don't restyle `<h3>`.
- Selection toolbars render **inside** `.grid` as `<SelectionToolbar>`. `.grid:has(.sel-toolbar)`
  reserves the bottom room automatically; never use `position: fixed`.

## Forms

- `<Field label hint error required>` wraps every input. Inside a card, group fields in
  `<div className="field-stack">` (16px gap).
- Two-column rows use `<div className="form-row">` (1fr 1fr, 14px gap, 6px top offset). Never
  free-type the gap.
- Action rows use `<div className="form-actions">` (right-aligned, 8px between, 16px top).

## Tables

- Wrap a plain `<table>` in `<TableShell>` (renders `<div className="table-shell">`) for the
  inset rhythm — rounded border, surface bg, `overflow: hidden` come from one class. Never
  hand-roll `border` / `border-radius` / `background` on the table itself.
- Headers use `<SortHeader>` (owns `aria-sort` + sort icon). Pagination renders below the shell
  in a `<TablePager>` hosting `<Pagination>` + rows-per-page `<Select>` — keep it as a
  **sibling** of `<TableShell>`, never inside it (the shell clips overflow and would crop the
  pager's focus rings).
- Monospace cells use `<Mac>` (or `className="mac"`) for MAC / IP / SKU columns. First-column
  avatar + label uses `<NameCell thumb={…}>`. Screen-reader-only captions use `<SrOnly>` (or
  `className="sr-only"`) — every `<table>` needs one for WCAG 1.3.1.

## Colour, type, icons

- Colours: tokens only. `var(--surface-1)`, `var(--text-1)`, `var(--text-2)`, `var(--border)`,
  `var(--brand)`, `var(--status-ok|warn|danger)`. **Never** literal hex / rgb / `#fff` / `#000`.
- Status copy: `<Pill variant="success|warn|danger|neutral">`, `<Badge>`, `<StatusIndicator>`.
  Never `<span style={{ color: 'green' }}>`.
- Type: prefer typed primitives (`<Display1>` / `<Display2>` / `<H1>`-`<H4>` / `<Body>` /
  `<BodyLg>` / `<Label>` / `<Caption>` / `<Eyebrow>` / `<Stat>` / `<Mono>` / `<Code>`); the
  underlying classes are `.dash-ui-display-1|2`, `.dash-ui-h1|2|3|4`, `.dash-ui-body|body-lg`,
  `.dash-ui-label|caption|eyebrow`, `.dash-ui-stat|mono|code` (note: `caption`, not `cap`). Use
  the class form only when you can't import a primitive. Inter for UI, JetBrains Mono for numeric.
- Icons: import from `@w5-ui/react` (23-icon canonical set). Size 16/20/24 only.

## Spacing — the only escape hatch

If a layout truly needs custom spacing, use `var(--space-1…12)` tokens. Never raw px in inline
styles. Radius: `--radius-md` (6 px, default), `--radius-lg` (12 px, modals), `--radius-pill`,
`--radius-app` (22 % for app squircles).

## Anti-patterns (auto-reject in code review)

- Inline `style={{ padding, gap, margin }}` on shell / grid / card elements
- Re-implementing `<Card>` / `<Stat>` / `<PageHeader>` as styled `<div>`s
- Hardcoded hex (`#1a1a1a`, `#006fff`, `#fff`)
- Wrapper `<div>` around `.grid`, or `display: flex; gap: …` on the page root
- `border-radius: 50%` on app-logo squircles (use `--radius-app`)
- `tokens.css` imported without `dashboard.css` (or without `styles.css`)

## When something looks "almost right"

Open the matching **Dashboard** story in Storybook, inspect the rendered DOM, and diff your
markup against it. The reference DOM is the single source of truth. If your class names differ,
that delta **is** the drift.
```

That block is intentionally framework-neutral on prose and React-flavoured in the snippets —
Svelte users swap component imports but every rule still holds.

## Self-check protocol (run before declaring "done")

When an agent finishes a userland page, it should run this checklist against the final markup
and DOM. Every miss is documented in **Foundations / Troubleshooting** with a one-line fix.

1. **`<html data-motif>` present** and set inline in `index.html` (not in a `useEffect`).
2. **`@w5-ui/<framework>/styles.css` imported exactly once** at the app entry. No bare
   `tokens.css` imports.
3. **One `<AppLayout>` at the app root.** No hand-rolled `.app` / `.workspace` / `<main>`.
4. **Page body is `<Banner>?` + `<PageHeader>` + `<div className="grid">`** as direct siblings.
   No wrapper around `.grid`. No `display: flex` on the page root.
5. **Every `.grid` child is `<Card>` or `<Stat>`** with `span ∈ {4, 6, 8, 12}`. No `span={5}`,
   no `span={7}`, no bare `<div>` lanes.
6. **No inline `style` for `padding` / `gap` / `margin`** on shell, header, or grid elements.
7. **No literal hex / rgb colour** in components. Grep the diff for `#[0-9a-f]{3,8}` and replace
   with `var(--…)` tokens.
8. **`<h3>` only inside `<Card>`.** `.card h3` carries the title rhythm; outside a card the
   selector doesn't apply.
9. **Forms use `.field-stack` / `.form-row` / `.form-actions`** — no bespoke flex containers.
10. **Computed style on root `.grid`:** `gap: 12px`, `padding: 16px 24px`. If either differs,
    a wrapper or override is in the way.

A passing run on those ten points reproduces the reference dashboard 1:1.

### Copy-paste DevTools probe

Open the running userland app, paste the snippet below into the browser console, and read the
report. It exercises six of the ten self-check points against the **live DOM** — the four
remaining (`styles.css` once, no inline shell `style`, `<h3>` only inside `<Card>`, forms use
`.field-stack` / `.form-row` / `.form-actions`) are diff-time checks that don't surface in the
computed style. Every failure prints the offending element so you can jump to it in Elements.

```js
;(() => {
  const fail = []
  const ok = []
  const note = (cond, msg, el) => (cond ? ok : fail).push(el ? { msg, el } : msg)

  const motif = document.documentElement.getAttribute('data-motif')
  note(motif === 'dark' || motif === 'light', `<html data-motif="${motif ?? 'unset'}">`)

  const probe = getComputedStyle(document.documentElement).getPropertyValue('--space-3').trim()
  note(probe === '12px', `--space-3 resolves to "${probe || 'unset'}" (expected 12px)`)

  const font = getComputedStyle(document.body).fontFamily
  note(/inter/i.test(font), `body font-family is "${font}" (expected Inter)`)

  const grid = document.querySelector('.grid')
  if (!grid) {
    fail.push('no `.grid` element found — page body is missing the canonical grid')
  } else {
    const cs = getComputedStyle(grid)
    note(cs.gap === '12px', `.grid gap is ${cs.gap} (expected 12px)`, grid)
    note(
      cs.paddingTop === '16px' && cs.paddingLeft === '24px',
      `.grid padding is ${cs.padding} (expected 16px 24px)`,
      grid,
    )
    const lanes = [...grid.children].filter(
      (c) => !c.classList.contains('card') && !c.classList.contains('sel-toolbar'),
    )
    note(lanes.length === 0, `.grid has ${lanes.length} non-Card child lane(s)`, lanes[0])
    const badSpans = [...grid.children].filter((c) => {
      const m = /span (\d+)/.exec(c.style.gridColumn || '')
      return m && ![4, 6, 8, 12].includes(Number(m[1]))
    })
    note(badSpans.length === 0, `${badSpans.length} card(s) with span ∉ {4,6,8,12}`, badSpans[0])
  }

  console.log(`%c✓ ${ok.length} pass`, 'color:#3ad07b', ok)
  console.log(`%c✗ ${fail.length} fail`, 'color:#ff6b6b', fail)
  return { ok, fail }
})()
```

Zero failures means the contract holds at the DOM level. Any failure points at the exact node
that drifts — paste the snippet again after each fix to confirm the regression collapses.

## Tailwind drift → typed primitive

The single biggest userland drift in practice: hand-rolled Tailwind class strings (`text-12
text-text-3`, `text-13 font-semibold text-text-1`, `text-[10px] font-semibold uppercase
tracking-[0.06em] text-text-3`) where a typed primitive already encodes the same contract.
Tailwind tokens line up with the type scale, but free-typed combinations re-derive what
`<Body>` / `<Label>` / `<Eyebrow>` already bundle — and as soon as one site uses 10px and
the next uses 11px, the rhythm splits. Reach for the primitive; if the page truly needs
something off-scale, surface it as a design fork before shipping.

> Chart drift in particular: every chart primitive ships as zero-dependency typed SVG that already
> matches the Dashboard chart rhythm (gridlines, padding, gradient stops, motif colours). Pulling
> in `recharts` / `chart.js` / `d3` re-derives the same picture with a 60 KB+ runtime, a different
> tooltip chrome, and a colour palette that drifts from `--color-brand` / `--color-status-*`. If
> the typed primitive truly can't express the chart, surface it as a design fork first — don't
> silently widen the dependency footprint.

The pattern: the typed primitive renders the **exact same DOM + class** that the Dashboard
reference uses, so userland tracks every future tweak (token shift, contrast bump, motion
preference) for free. Tailwind utilities remain the right tool for one-off layout glue
(`flex`, `gap-3`, `min-w-0`, `overflow-hidden`) — just don't re-derive typography or the
canonical chrome with them.

## Why an agent brief at all?

Documentation that works for humans assumes the reader can navigate sixteen MDX pages, follow
cross-references, and skim for the relevant section. An agent's prompt budget rewards the
opposite: **one** flat directive list, no chrome, every rule actionable from its own line. The
brief above is sized to fit comfortably inside a project-level `AGENTS.md` alongside whatever
else the project needs. The Foundations pages remain the long-form source of truth — this page
is just the executive summary an agent can keep in working memory.
