# Setup

This page is the **end-to-end recipe** for getting a fresh userland app to render like the
Dashboard story. Follow it top-to-bottom once. Skip ahead and a single missing line (the styles
import, the `data-motif` attribute, an opinionated body reset) silently re-introduces all of the
spacing and contrast drift the design system was meant to eliminate.

> If your userland looks "almost right but the gaps are off and the colours are washed out", you
> almost certainly missed exactly one step below. Re-read in order, don't cherry-pick.

## 1. Install

```bash
yarn add @w5-ui/react @w5-ui/tokens @w5-ui/assets
```

`@w5-ui/react` already depends on `@w5-ui/tokens` and `@w5-ui/assets` transitively — listing them
explicitly is optional, but it makes deduping in monorepos predictable.

| Package         | What it ships                                                       |
| --------------- | ------------------------------------------------------------------- |
| `@w5-ui/react`  | Components, `styles.css` (re-exports tokens + dashboard CSS)        |
| `@w5-ui/tokens` | `tokens.css` (motif tokens), `dashboard.css` (`.app`/`.grid`/`.card`)|
| `@w5-ui/assets` | Brand SVGs (logo, product marks) consumed by `Topbar`               |

Svelte users swap the first row for `@w5-ui/svelte`. The remaining two packages are framework-
agnostic — never install them per-app twice.

## 2. The HTML shell

Set `data-motif` on `<html>` **before the bundle parses**. This is the single source of truth for
every semantic colour, border, and shadow in the system.

```html
<!-- index.html -->
<!doctype html>
<html lang="en" data-motif="dark">
  <head>
    <meta charset="UTF-8" />
    <meta name="viewport" content="width=device-width, initial-scale=1.0" />
    <title>Acme Console</title>
  </head>
  <body>
    <div id="root"></div>
    <script type="module" src="/src/main.tsx"></script>
  </body>
</html>
```

That's it for the shell. Don't add `body { background: #fff }`, don't add `html { height: 100% }`,
don't wrap `#root` in a flex container — `AppLayout` already owns the viewport grid.

## 3. The entry (React)

Import the styles **once**, at the very top of your entry, before the first component import.

```tsx
// src/main.tsx
import React from 'react'
import ReactDOM from 'react-dom/client'
import '@w5-ui/react/styles.css' // re-exports @w5-ui/tokens/tokens.css + dashboard.css
import { App } from './App.js'

ReactDOM.createRoot(document.getElementById('root')!).render(
  <React.StrictMode>
    <App />
  </React.StrictMode>,
)
```

Svelte:

```ts
// src/main.ts
import '@w5-ui/svelte/styles.css'
import App from './App.svelte'

new App({ target: document.getElementById('root')! })
```

Two things go wrong here often enough to call out:

- **Duplicate imports.** If you also `import '@w5-ui/tokens/tokens.css'` somewhere later, the
  cascade order changes and component overrides (e.g. `Card` corner radii) lose to the raw token
  layer. Import once via `styles.css` and stop there.
- **Late imports.** Importing `styles.css` inside a feature module makes its rules win against the
  components — `@layer` order is decided by first-encounter. Keep it at the entry.

## 4. The app root

Render `AppLayout` exactly once. The dashboard chrome (skip link, topbar, sidebar, sticky page
header, `<main>`) lives inside it. Pages render as its `children`.

```tsx
// src/App.tsx
import { AppLayout, Topbar, Sidebar, type SidebarSectionDef } from '@w5-ui/react'
import { useState } from 'react'
import { Dashboard } from './pages/Dashboard.js'

const SECTIONS: SidebarSectionDef[] = [
  /* ...see Foundations / Navigation for the canonical shape... */
]

export function App() {
  const [page, setPage] = useState('dashboard')
  return (
    <AppLayout
      pageLabel="Dashboard"
      topbar={<Topbar siteName="HQ" activeApp="system" />}
      sidebar={<Sidebar sections={SECTIONS} activeId={page} onChange={setPage} />}
    >
      {page === 'dashboard' && <Dashboard />}
      {/* ...other pages... */}
    </AppLayout>
  )
}
```

Inside a page, the canonical body is `Banner` → `PageHeader` → `.grid` of `Card` / `Stat`. See
**Foundations / Page layout** for the full skeleton and grid math.

## 5. Anti-FOUC: pick the motif before paint (optional)

Skip if your app is dark-only. If users can flip light/dark, inline a tiny script in `index.html`
that sets `data-motif` before the React bundle loads.

```html
<script>
  ;(function () {
    var m = localStorage.getItem('motif')
    if (m !== 'dark' && m !== 'light') {
      m = matchMedia('(prefers-color-scheme: light)').matches ? 'light' : 'dark'
    }
    document.documentElement.setAttribute('data-motif', m)
  })()
</script>
```

Then expose a `useMotif()` hook (or Svelte action) that mirrors writes back to
`localStorage`. The pattern lives in **Foundations / Motifs**. Doing this inside a `useEffect`
instead of an inline script causes a one-frame flash of the wrong motif on every refresh.

## 6. Fonts

`@w5-ui/tokens` declares **Inter** for body and **JetBrains Mono** for tabular text (IPs, MACs,
SKUs). Self-host or load via Google Fonts in `index.html`:

```html
<link rel="preconnect" href="https://fonts.gstatic.com" crossorigin />
<link
  rel="stylesheet"
  href="https://fonts.googleapis.com/css2?family=Inter:wght@400;500;600;700&family=JetBrains+Mono:wght@400;500&display=swap"
/>
```

If a custom system font is preferred, override `--dash-ui-font-sans` / `--dash-ui-font-mono` once on
`:root` — never per component. See **Foundations / Type**.

## 7. Tailwind (optional)

Userland that already uses Tailwind v4 can pull in the token utilities and the motif-aware
`dark:` variant in one import:

```css
/* src/app.css */
@import 'tailwindcss';
@import '@w5-ui/tokens/tailwind.css';
```

That file declares
`@custom-variant dark (&[data-motif='dark'], [data-motif='dark'] &)`, so `dark:bg-bg-1` already
tracks the motif attribute — no `darkMode: 'class'` / `'media'` override needed.

```tsx
<div className="bg-bg-1 dark:bg-bg-2">{/* … */}</div>
```

See **Foundations / Motifs** for the full rationale and **Foundations / Colours** for the token
class names (`bg-bg-*`, `text-text-*`, `border-border-*`, `bg-status-*-bg`, …).

## 8. Sanity checklist

Open the running app next to the **Dashboard** story in Storybook. Each row below should match.
If one doesn't, the fix is in the named foundations page — don't tweak in userland.

| Check                                                              | If it fails, read…                |
| ------------------------------------------------------------------ | --------------------------------- |
| Page background is the deep dark surface, not pure black or white  | Foundations / Motifs              |
| `<main>` has 16 px top / 24 px side padding around the grid        | Foundations / Page layout         |
| Cards have 6 px radius, 16 px padding, 12 px gap between them      | Foundations / Cards               |
| `Stat span={4}` cards line up 3-across, 8 + 4 lines up 2-across    | Foundations / Page layout (grid)  |
| `<h3>` inside a `<Card>` renders 14 px / weight 500 / `text-1`     | Foundations / Type                |
| Pills render rounded full, status-tinted, never bare hex colours   | Foundations / Status & feedback   |
| Sidebar links highlight on hover with `--row-hover`, not a custom  | Foundations / Navigation          |
| `body`, `#root`, and the page wrapper have **no** margin or padding| This page (step 2)                |

If three or more rows fail at once, suspect the styles import (step 3) or the motif attribute
(step 2). Both fix the symptoms in bulk.

## Common mistakes (and the fix)

| Symptom                                                                 | Likely cause                                                                | Fix                                                                                       |
| ----------------------------------------------------------------------- | --------------------------------------------------------------------------- | ----------------------------------------------------------------------------------------- |
| Bundle loads, screen is blank white, no errors                          | `@w5-ui/react/styles.css` not imported in the entry                         | Add it as the **first** import in `main.tsx`                                              |
| Components render but colours look raw — neutral greys, blue link blue  | `data-motif` missing from `<html>`, tokens fall through to default `:root`  | Add `data-motif="dark"` to `<html>` (step 2). See **Foundations / Motifs**                |
| Card padding looks fine but the page edges feel too tight / too wide    | A custom `<div style={{ padding: ... }}>` wraps the page below `<main>`     | Render `Banner` / `PageHeader` / `.grid` as siblings of `<main>` — see Page layout        |
| Gaps between cards look wrong (8 px or 16 px instead of 12 px)          | Re-implementing `.grid` with `display: flex; gap: 1rem`                     | Use `<div className="grid">` from the dashboard CSS — `repeat(12, 1fr)` + 12 px gap       |
| Cards render but no shadow / depth                                      | Imported only `tokens.css`, not `dashboard.css`                             | Use `@w5-ui/react/styles.css` (step 3) — it bundles both                                  |
| Two scrollbars, page won't scroll the content area                      | `html, body { height: 100% }` plus `#root { display: flex }`                | Delete those resets. `AppLayout` already sets up the viewport grid                        |
| Topbar / Sidebar render twice when navigating                           | `AppLayout` rendered inside a page component instead of once at App root    | Move `AppLayout` to `App.tsx`; pages render as `children`                                 |
| Tailwind `dark:` classes never trigger                                  | Tailwind config sets `darkMode: 'media'` / `'class'` and beats the variant  | Remove the override; `@w5-ui/tokens/tailwind.css` already binds `dark:` to `data-motif`   |
| First paint flashes the wrong motif on refresh                          | Motif read happens in a `useEffect` after the bundle parses                 | Move it to an inline `<script>` in `index.html` (step 5)                                  |
| Fonts look like the system default, not Inter                           | Font link tag missing or `font-family` overridden somewhere in userland CSS | Add the `<link>` (step 6); don't set `body { font-family: ... }`                          |
| Status pills are the right colour in dark but garish in light           | Hardcoded `bg-red-500/10` reproduces the tint manually                      | Use `bg-status-danger-bg` / `text-status-danger` — both flip per motif                    |
