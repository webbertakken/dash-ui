# Dash UI

A multi-framework dashboard design system, exposed as **React**, **Svelte**, and **Web Component**
packages.

## Packages

| Package         | Description                                        |
| --------------- | -------------------------------------------------- |
| `@w5-ui/tokens` | CSS variables (light + dark motifs) + JS token map |
| `@w5-ui/assets` | Brand SVG logos + product marks                    |
| `@w5-ui/react`  | 168 React components                               |
| `@w5-ui/svelte` | 168 Svelte components (mirrored API)               |
| `@w5-ui/wc`     | Web Components compiled from the Svelte source     |

## Visual identity

- **Brand blue**: `#006FFF` (the only saturated colour in the dashboard motif)
- **Status**: `#00B070` ok, `#F5A623` warn, `#F03A3A` critical
- **Type**: Inter (UI) + JetBrains Mono (IPs / MACs / SKUs)
- **Default radius**: 6 px (12 for modals, 9999 for pills)

Two motifs ship in `tokens.css`:

```html
<html data-motif="dark">
  <!-- the default; swap to data-motif="light" to flip surfaces, text and borders -->
</html>
```

## Where to start

Userland renders the dashboard correctly when **three** things are in place: the styles import, the
`data-motif` attribute, and the canonical `AppLayout` → `PageHeader` → `.grid` → `Card` shell. If a
new userland app looks "almost right but the spacing is off", one of those three is missing.

Pick the entry point that matches your situation:

| Situation                                                       | Read first                  |
| --------------------------------------------------------------- | --------------------------- |
| Wiring a fresh React or Svelte app                              | **Foundations / Setup**     |
| Already importing `styles.css`, page chrome still looks off     | **Foundations / Page layout** |
| Userland already uses Tailwind v4                               | **Foundations / Setup §7** + **Motifs** |
| Light/dark toggle flashes the wrong motif on refresh            | **Foundations / Motifs**    |
| Spacing between cards / sections looks wrong                    | **Foundations / Spacing** + **Page layout** |
| Hand-rolling a status colour or hex                             | **Foundations / Colours**   |
| Userland looks "almost right" — fast symptom-to-fix index       | **Foundations / Troubleshooting** |

Reproduce the recipe in **Setup** and the skeleton in **Page layout** verbatim — spacing, rhythm,
and contrast then match the Dashboard story without hand-tuning.

## Userland recipe in one screen

```tsx
// src/main.tsx
import '@w5-ui/react/styles.css' // tokens + dashboard.css in one import
import { AppLayout, Topbar, Sidebar, PageHeader, Card, Stat } from '@w5-ui/react'

<AppLayout pageLabel="Dashboard" topbar={<Topbar siteName="HQ" />} sidebar={<Sidebar … />}>
  <PageHeader title="Dashboard" />
  <div className="grid">
    <Stat span={4} label="Internet" value="847" unit="Mbps" />
    <Card span={8}>…</Card>
  </div>
</AppLayout>
```

Add `data-motif="dark"` to `<html>` and that single snippet is the whole contract. Anything else
(custom paddings, ad-hoc wrappers, raw hex colours) is what drags userland away from the reference.
See **Foundations / Setup** for the full walkthrough.
