# Dash UI

A multi-framework dashboard design system, exposed as **React**, **Svelte**, and **Web Component**
packages.

## Packages

| Package         | Description                                        |
| --------------- | -------------------------------------------------- |
| `@w5-ui/tokens` | CSS variables (light + dark motifs) + JS token map |
| `@w5-ui/assets` | Brand SVG logos + product marks                    |
| `@w5-ui/react`  | 168 React components                               |
| `@w5-ui/svelte` | 168 Svelte components (mirrored API)               |
| `@w5-ui/wc`     | Web Components compiled from the Svelte source     |

## Visual identity

- **Brand blue**: `#006FFF` (the only saturated colour in the dashboard motif)
- **Status**: `#00B070` ok, `#F5A623` warn, `#F03A3A` critical
- **Type**: Inter (UI) + JetBrains Mono (IPs / MACs / SKUs)
- **Default radius**: 6 px (12 for modals, 9999 for pills)

Two motifs ship in `tokens.css`:

```html
<html data-motif="dark">
  <!-- the default -->
  <html data-motif="light"></html>
</html>
```

## Sidebar layout

Components are grouped into 12 categories (max), the same set as the Svelte and Web Components
Storybooks at `/svelte/` and `/wc/`.

## Userland recipe

If you are wiring `@w5-ui/react` (or `@w5-ui/svelte`) into your own app, start with **Foundations /
Setup** for the end-to-end install + entry-file walkthrough, then read **Foundations / Page
layout** for the `Banner` → `PageHeader` → `.grid` → `Card` skeleton that every reference page
uses. Reproduce both verbatim and spacing, rhythm, and contrast match the Dashboard story without
hand-tuning.
