# Recipes

Complete, copy-paste page templates for Svelte 5 userland. Each recipe is a working file — drop it
into a fresh SvelteKit / Vite-Svelte app, import `@w5-ui/svelte/styles.css`, set
`<html data-motif="dark">`, and it renders pixel-for-pixel like the matching Dashboard story.

If you are wiring a brand-new userland app, copy from here first; tune later. The recipes
deliberately favour the canonical components (`AppLayout`, `PageHeader`, `Card`, `Stat`,
`TableShell`, `FieldStack`) over hand-rolled markup — that is the difference between userland that
matches and userland that drifts.

> Every recipe assumes `@w5-ui/svelte/styles.css` is imported once at the app entry point and
> `<html data-motif="dark">` is set inline in `index.html`. See **Foundations / Setup** for the
> entry-file walkthrough.

## Recipe 1 — Dashboard overview

A landing page: hero stats across the top, two wide charts underneath, one full-width tile at the
bottom. Mirrors the **Dashboard** story tile-for-tile.

```svelte
<!-- src/routes/+page.svelte -->
<script lang="ts">
  import {
    AppLayout,
    Topbar,
    Sidebar,
    PageHeader,
    Stat,
    Card,
    CardTitle,
    Grid,
    Sparkline,
    LineChart,
    BarChart,
  } from '@w5-ui/svelte'

  let activeApp = $state('system')
  let activePage = $state('dashboard')
  const throughput = [/* ...time-series points... */]
  const top = [/* ...sparkline values... */]
  const traffic = [/* ...bar-chart points... */]
</script>

<AppLayout pageLabel="Dashboard">
  {#snippet topbar()}<Topbar siteName="HQ" bind:activeApp />{/snippet}
  {#snippet sidebar()}<Sidebar sections={SECTIONS} bind:activeId={activePage} />{/snippet}

  <PageHeader title="Dashboard" />
  <Grid>
    <Stat
      span={4}
      label="Internet"
      value="847"
      unit="Mbps · down"
      sub="18 ms · 0.0% loss"
      delta="+12% 24h"
      deltaDir="up"
    />
    <Stat
      span={4}
      label="Clients"
      value="312"
      unit="active"
      sub="42 wired · 270 wireless"
      delta="+4 24h"
      deltaDir="up"
    />
    <Stat
      span={4}
      label="Alerts"
      value="3"
      unit="open"
      sub="1 critical · 2 warn"
      delta="-2 24h"
      deltaDir="down"
    />

    <Card span={8}>
      <CardTitle>Throughput</CardTitle>
      <LineChart data={throughput} />
    </Card>
    <Card span={4}>
      <CardTitle>Top clients</CardTitle>
      <Sparkline data={top} />
    </Card>

    <Card span={12}>
      <CardTitle>Traffic by device</CardTitle>
      <BarChart data={traffic} />
    </Card>
  </Grid>
</AppLayout>
```

**What carries the look**: `AppLayout` owns the shell padding. `PageHeader` owns the sticky header
rhythm. `<Grid>` owns the 12-column gutter (typed equivalent of `<div class="grid">`). `Stat` and
`<Card span={n}>` own tile padding; `<CardTitle>` is the typed sibling of `<h3>` so the `.card h3`
rhythm survives codegen, MDX, or stray wrapper `<div>`s. Nothing else needs to be set.

> `bind:activeApp` and `bind:activeId` are required because both props are `$bindable()`. Without
> the `bind:` directive the parent never sees the user's selection and the sidebar / app-switcher
> never highlights.

## Recipe 2 — List page with table

Devices-style page: page header with primary action + search, table with sortable headers,
pagination underneath, bottom selection toolbar when rows are checked. The selection toolbar
triggers `.grid:has(.sel-toolbar)` automatically — no extra padding required.

```svelte
<!-- src/routes/devices/+page.svelte -->
<script lang="ts">
  import {
    AppLayout,
    Topbar,
    Sidebar,
    PageHeader,
    Card,
    Grid,
    Button,
    SearchBox,
    Table,
    SortHeader,
    TableShell,
    Pagination,
    SelectionToolbar,
    Checkbox,
  } from '@w5-ui/svelte'
  import DeviceRow from './DeviceRow.svelte'

  let selected = $state<string[]>([])
  let page = $state(1)
  const rows = [/* ...devices... */]
  const total = rows.length
</script>

<AppLayout pageLabel="Devices">
  {#snippet topbar()}<Topbar siteName="HQ" activeApp="system" />{/snippet}
  {#snippet sidebar()}<Sidebar sections={SECTIONS} activeId="devices" />{/snippet}

  {#snippet actions()}
    <SearchBox placeholder="Search devices…" />
    <Button variant="primary">Adopt</Button>
  {/snippet}
  <PageHeader title="Devices" {actions} />

  <Grid>
    <Card span={12}>
      <TableShell>
        <Table>
          <thead>
            <tr>
              <th scope="col"><Checkbox /></th>
              <SortHeader sort="asc">Name</SortHeader>
              <th scope="col">Model</th>
              <th scope="col" class="mac">MAC</th>
              <SortHeader>Clients</SortHeader>
            </tr>
          </thead>
          <tbody>
            {#each rows as r (r.id)}
              <DeviceRow row={r} />
            {/each}
          </tbody>
        </Table>
      </TableShell>
      <Pagination {page} {total} onchange={(p) => (page = p)} />
    </Card>

    {#if selected.length > 0}
      <SelectionToolbar count={selected.length} onclear={() => (selected = [])}>
        <Button>Move</Button>
        <Button variant="danger">Forget</Button>
      </SelectionToolbar>
    {/if}
  </Grid>
</AppLayout>
```

**What carries the look**: `<Grid>` is the typed page grid. `<TableShell>` wraps `<Table>` for the
inset rhythm. `<th scope="col">` carries a11y. `SortHeader` owns the sort icon + `aria-sort`. The
selection toolbar is rendered _inside_ `<Grid>` so the `.grid:has(.sel-toolbar)` rule reserves room
for it — never as a sibling of `<main>` or a fixed-positioned overlay you styled yourself.

## Recipe 3 — Settings / form page

Two-column form with grouped sections. `<FieldStack>` owns the 16 px vertical rhythm, `<FormRow>`
owns the 14 px column gap, `<FormActions>` owns the right-aligned button row. Drop in
`<Field label hint error required>` for every input.

```svelte
<!-- src/routes/settings/+page.svelte -->
<script lang="ts">
  import {
    AppLayout,
    Topbar,
    Sidebar,
    PageHeader,
    Card,
    CardTitle,
    Field,
    FieldStack,
    FormRow,
    FormActions,
    Input,
    Select,
    Toggle,
    Button,
    Callout,
  } from '@w5-ui/svelte'

  let name = $state('HQ')
  let country = $state('NL')
  let timezone = $state('Europe/Amsterdam')
  let maint = $state(true)
  let cidr = $state('10.0.0.0/24')
  let cidrErr = $state<string | null>(null)
  let mtu = $state(1500)

  function onSave(e: SubmitEvent) {
    e.preventDefault()
    /* ...persist... */
  }
</script>

<AppLayout pageLabel="Settings">
  {#snippet topbar()}<Topbar siteName="HQ" />{/snippet}
  {#snippet sidebar()}<Sidebar sections={SECTIONS} activeId="settings" />{/snippet}

  {#snippet actions()}
    <Button variant="primary" type="submit" form="settings">Save</Button>
  {/snippet}
  <PageHeader title="Settings" {actions} />

  <form id="settings" class="grid" onsubmit={onSave}>
    <Card span={6}>
      <CardTitle>Site</CardTitle>
      <FieldStack>
        <Field label="Site name" required hint="Visible in the topbar">
          <Input bind:value={name} />
        </Field>
        <FormRow>
          <Field label="Country"><Select bind:value={country} /></Field>
          <Field label="Timezone"><Select bind:value={timezone} /></Field>
        </FormRow>
        <Field label="Maintenance window" hint="Daily 04:00 local">
          <Toggle bind:checked={maint} />
        </Field>
      </FieldStack>
    </Card>

    <Card span={6}>
      <CardTitle>Network</CardTitle>
      <Callout variant="info" title="Restart required">
        Changes to LAN subnet require a controller reboot.
      </Callout>
      <FieldStack>
        <Field label="LAN subnet" error={cidrErr}>
          <Input bind:value={cidr} />
        </Field>
        <Field label="MTU"><Input bind:value={mtu} /></Field>
      </FieldStack>
      <FormActions>
        <Button>Cancel</Button>
        <Button variant="primary" type="submit">Apply</Button>
      </FormActions>
    </Card>
  </form>
</AppLayout>
```

**What carries the look**: the `<form class="grid">` _is_ the grid — no extra wrapper. `<CardTitle>`
keeps the section-title rhythm under the `.card h3` rule without leaking a raw `<h3>` into
copy-paste templates; `<FieldStack>` ladders fields with 16 px gaps; `<FormRow>` two-columns inputs
without bespoke gap overrides; `<FormActions>` right-aligns and adds the 16 px top offset. Replace
these typed primitives with hand-rolled flex containers and the rhythm drifts within one PR. The
underlying CSS hooks (`.card h3`, `.field-stack`, `.form-row`, `.form-actions`) ship for parity, but
the typed primitives are the preferred path — they are auto-importable and survive a class-name
rename without touching userland.

## Empty-state / loading shell

When data is still loading or empty, render the same shell so layout doesn't jump:

```svelte
<script lang="ts">
  import { AppLayout, PageHeader, Grid, Card, Skeleton, EmptyState, Table } from '@w5-ui/svelte'
  let loading = $state(true)
  let items = $state<unknown[]>([])
</script>

<AppLayout pageLabel="Devices">
  {#snippet topbar()}<Topbar /> {/snippet}
  {#snippet sidebar()}<Sidebar /> {/snippet}
  <PageHeader title="Devices" />
  <Grid>
    <Card span={12}>
      {#if loading}
        <Skeleton lines={6} />
      {:else if items.length === 0}
        <EmptyState title="No devices yet" hint="Adopt one to get started" />
      {:else}
        <Table><!-- ... --></Table>
      {/if}
    </Card>
  </Grid>
</AppLayout>
```

`<Skeleton>` and `<EmptyState>` both reserve the same vertical room as the eventual table, so the
page never jolts when data arrives. Never conditionally render the `<Card>` itself — the grid lane
goes with it and your other tiles reflow.

## What every recipe omits (on purpose)

- **No inline `style="padding: ...; gap: ..."`.** Everything spatial comes from `.grid` / `.card` /
  `.field-stack` / `.form-row` / `.form-actions`.
- **No raw hex colours.** Status comes from `Pill` / `Callout` / `Banner`; copy comes from
  `var(--text-1)` and `var(--text-2)`.
- **No bespoke page header.** `PageHeader` (or the `.ph-bar` / `.ph-title` / `.ph-actions`
  primitives) owns the sticky border + padding.
- **No wrapper `<div>` around `.grid`.** The grid is a direct child of `<main>` (which `AppLayout`
  renders for you). Wrappers nullify the 16 × 24 page gutter.

If a recipe needs to bend one of those rules, that delta belongs in **Foundations** as a new
primitive — not as a one-off override in userland. Surface it instead of hand-rolling.
