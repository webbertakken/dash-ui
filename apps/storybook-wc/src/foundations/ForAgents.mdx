# For agents

A single condensed brief for **agentic implementers** (Claude Code, Cursor, Copilot, etc.) wiring a
Web Components userland app on top of `@w5-ui/wc`. The other Foundations pages explain **why** the
contract is shaped the way it is; this page is the **what** in directive form, sized to drop
straight into a userland `AGENTS.md` / `CLAUDE.md` / Cursor rules file.

> If you are a human, skim this once and bookmark **Foundations / Setup** + **Recipes**. The wording
> below is deliberately terse so an LLM can keep the whole contract in its working set.

## Paste-into-userland brief

Copy this block verbatim into `AGENTS.md` (or `CLAUDE.md`, or `.cursor/rules/dash-ui.md`) at the
root of any Web Components userland project that consumes `@w5-ui/wc`. It is written as imperative
rules — keep it short so it stays inside the model's prompt budget.

```md
# Dash UI — Web Components implementation contract

Userland renders like the reference dashboard **only** when these rules hold. Every visual drift
("spacing feels off", "colours look washed out", "elements render as empty boxes") maps to a rule
below.

## Setup (do once)

- Import `@w5-ui/wc/styles.css` **once** at the app entry. Never import `@w5-ui/tokens/tokens.css`
  alone — `dashboard.css` is what wires `.app` / `.grid` / `.card`.
- Register the elements: `import '@w5-ui/wc'` at the entry. Without it every `<uni-*>` is an
  `HTMLUnknownElement` and renders as an empty inline box.
- Set `<html data-motif="dark">` (or `"light"`) **inline in `index.html`**, never after the bundle
  parses. Anti-FOUC requires the attribute before the first paint.
- Load Inter (UI) + JetBrains Mono (numeric cells). System fonts shift the type scale ~1px.

## Page shell (every page, no exceptions)

- Compose the shell with raw `.app` / `.workspace` / `.content` CSS hooks. Nest `<uni-topbar>` and
  `<uni-sidebar>` at the matching positions. `<uni-app-layout>` exposes only the default slot, so
  Topbar / Sidebar do **not** travel through it.
- Inside `.content`, render at most three siblings: optional `<uni-banner>`, then
  `<uni-page-header>`, then `.grid` (or `<uni-grid>`). No extra wrapper around the grid.
- `<uni-page-header title="…">` owns the sticky header. Slot custom actions into the `actions` named
  slot; never re-implement `.ph-bar` / `.ph-title` / `.ph-actions` by hand.

## Attribute vs property

- **Primitive props** (string / number / boolean): pass as HTML attributes (`<uni-stat span="4">`,
  `<uni-toggle checked>`).
- **Complex props** (array, object, callback, Date): set as JS properties after `querySelector` /
  `createElement`. Attributes serialise to strings only and will render `[object Object]` or trigger
  silent no-ops.
- **Events**: every `uni-*` element dispatches plain `CustomEvent`s — wire with
  `el.addEventListener('change', (e) => …)`. Don't try to pass `onchange={…}` as an attribute.

## Inside the grid

- `.grid` is `repeat(12, 1fr)`, 12 px gap, 16 px × 24 px padding. **Never** override `gap` /
  `padding` on `.grid` or wrap it.
- Every direct child is `<uni-card span="n">` or `<uni-stat span="n">`. Use `span` in {4, 6, 8, 12}
  — never 5 or 7.
- For a hero number, use `<uni-stat>` (it _is_ a `<uni-card>`). For richer tiles, compose inside
  `<uni-card>` and let `.card h3` style the title — don't restyle `<h3>`.
- Selection toolbars render **inside** `.grid` as `<uni-selection-toolbar>`.
  `.grid:has(.sel-toolbar)` reserves the bottom room automatically; never use `position: fixed`.

## Forms

- `<uni-field label hint error required>` wraps every input. Inside a card, group fields in
  `<div class="field-stack">` (16 px gap).
- Two-column rows use `<div class="form-row">` (1fr 1fr, 14 px gap, 6 px top offset). Never
  free-type the gap.
- Action rows use `<div class="form-actions">` (right-aligned, 8 px between, 16 px top).
- Two-way binding: listen to the element's `change` event and assign the new value back to your
  state. The DOM is the source of truth.

## Tables

- Wrap a plain `<table>` in `<uni-table-shell>` for the inset rhythm — rounded border, surface bg,
  `overflow: hidden` come from one class. Never hand-roll `border` / `border-radius` / `background`
  on the table itself.
- Headers use `<uni-sort-header>` (owns `aria-sort` + sort icon). Pagination renders below the shell
  in a `<uni-table-pager>` hosting `<uni-pagination>` + rows-per-page `<uni-select>` — keep it as a
  **sibling** of `<uni-table-shell>`, never inside it.
- Monospace cells use `<uni-mac>` (or `class="mac"`) for MAC / IP / SKU columns. First-column
  avatar + label uses `<uni-name-cell>`. Screen-reader-only captions use `<uni-sr-only>` (or
  `class="sr-only"`) — every `<table>` needs one for WCAG 1.3.1.

## Colour, type, icons

- Colours: tokens only. `var(--surface-1)`, `var(--text-1)`, `var(--text-2)`, `var(--border)`,
  `var(--brand)`, `var(--status-ok|warn|danger)`. **Never** literal hex / rgb / `#fff` / `#000`.
- Status copy: `<uni-pill variant="success|warn|danger|neutral">`, `<uni-badge>`,
  `<uni-status-indicator>`. Never `<span style="color: green">`.
- Type: prefer typed primitives (`<uni-display1>` / `<uni-display2>` / `<uni-h1>`-`<uni-h4>` /
  `<uni-body>` / `<uni-body-lg>` / `<uni-label>` / `<uni-caption>` / `<uni-eyebrow>` / `<uni-stat>`
  / `<uni-mono>` / `<uni-code>`). The underlying classes are `.dash-ui-display-1|2`,
  `.dash-ui-h1|2|3|4`, `.dash-ui-body|body-lg`, `.dash-ui-label|caption|eyebrow`,
  `.dash-ui-stat|mono|code`. Use the class form only when you can't reach a primitive. Inter for UI,
  JetBrains Mono for numeric.
- Icons: `<uni-search-icon>`, `<uni-bell-icon>`, etc. (23-icon canonical set). Size 16 / 20 / 24
  only via the `size` attribute.

## Spacing — the only escape hatch

If a layout truly needs custom spacing, use `var(--space-1…12)` tokens. Never raw px in inline
styles. Radius: `--radius-md` (6 px, default), `--radius-lg` (12 px, modals), `--radius-pill`,
`--radius-app` (22 % for app squircles).

## Anti-patterns (auto-reject in code review)

- `<uni-*>` element without the side-effect `import '@w5-ui/wc'` at the entry
- Complex props (`apps`, `sections`, callbacks, dates) passed as HTML attributes
- Inline `style="padding: ...; gap: ...; margin: ..."` on shell / grid / card elements
- Re-implementing `<uni-card>` / `<uni-stat>` / `<uni-page-header>` as styled `<div>`s
- Hardcoded hex (`#1a1a1a`, `#006fff`, `#fff`)
- Wrapper `<div>` around `.grid`, or `display: flex; gap: …` on the page root
- `border-radius: 50%` on app-logo squircles (use `--radius-app`)
- `tokens.css` imported without `dashboard.css` (or without `styles.css`)

## When something looks "almost right"

Open the matching **Dashboard** story in Storybook, inspect the rendered DOM, and diff your markup
against it. The reference DOM is the single source of truth. If your class names or element tags
differ, that delta **is** the drift.
```

That block is intentionally WC-specific on the attribute-vs-property contract and the side-effect
register — both are pitfalls that React / Svelte userland never hits and that the React-flavoured
brief does not mention.

## Self-check protocol (run before declaring "done")

When an agent finishes a userland page, it should run this checklist against the final markup and
DOM. Every miss is documented in **Foundations / Troubleshooting** with a one-line fix.

1. **`<html data-motif>` present** and set inline in `index.html`.
2. **`@w5-ui/wc/styles.css` imported exactly once** at the app entry. No bare `tokens.css` imports.
3. **`import '@w5-ui/wc'` runs at the entry.** Verify every `<uni-*>` element on the page has
   children rendered, not an empty inline box.
4. **One `.app` shell at the page root.** Inside it: `<uni-topbar>` → `.workspace` →
   (`<uni-sidebar>` + `<main class="content">`).
5. **Page body is `<uni-banner>?` + `<uni-page-header>` + `.grid`** as direct siblings. No wrapper
   around `.grid`. No `display: flex` on the page root.
6. **Every `.grid` child is `<uni-card>` or `<uni-stat>`** with `span ∈ {4, 6, 8, 12}`. No
   `span="5"`, no `span="7"`, no bare `<div>` lanes.
7. **No inline `style` for `padding` / `gap` / `margin`** on shell, header, or grid elements.
8. **No literal hex / rgb colour** in markup. Grep the diff for `#[0-9a-f]{3,8}` and replace with
   `var(--…)` tokens.
9. **Complex props set as JS properties**, not HTML attributes. Grep the diff for
   `<uni-\w+\s+(apps|sections|items|options|onChange|onAction)=` and lift those to JS-side property
   assignments.
10. **Computed style on root `.grid`:** `gap: 12px`, `padding: 16px 24px`. If either differs, a
    wrapper or override is in the way.

A passing run on those ten points reproduces the reference dashboard 1:1.

### Copy-paste DevTools probe

Open the running userland app, paste the snippet below into the browser console, and read the
report. It exercises seven of the ten self-check points against the **live DOM** — the three
remaining (`styles.css` once, no inline shell `style`, forms use `.field-stack` / `.form-row` /
`.form-actions`) are diff-time checks that don't surface in the computed style. Every failure prints
the offending element so you can jump to it in Elements.

```js
;(() => {
  const fail = []
  const ok = []
  const note = (cond, msg, el) => (cond ? ok : fail).push(el ? { msg, el } : msg)

  const motif = document.documentElement.getAttribute('data-motif')
  note(motif === 'dark' || motif === 'light', `<html data-motif="${motif ?? 'unset'}">`)

  const probe = getComputedStyle(document.documentElement).getPropertyValue('--space-3').trim()
  note(probe === '12px', `--space-3 resolves to "${probe || 'unset'}" (expected 12px)`)

  const font = getComputedStyle(document.body).fontFamily
  note(/inter/i.test(font), `body font-family is "${font}" (expected Inter)`)

  // Every uni-* element must be registered (i.e. an upgraded custom element).
  const unupgraded = [...document.querySelectorAll('*')].filter(
    (el) =>
      el.tagName.toLowerCase().startsWith('uni-') && !customElements.get(el.tagName.toLowerCase()),
  )
  note(
    unupgraded.length === 0,
    `${unupgraded.length} unupgraded uni-* element(s) — missing register?`,
    unupgraded[0],
  )

  const grid = document.querySelector('.grid')
  if (!grid) {
    fail.push('no `.grid` element found — page body is missing the canonical grid')
  } else {
    const cs = getComputedStyle(grid)
    note(cs.gap === '12px', `.grid gap is ${cs.gap} (expected 12px)`, grid)
    note(
      cs.paddingTop === '16px' && cs.paddingLeft === '24px',
      `.grid padding is ${cs.padding} (expected 16px 24px)`,
      grid,
    )
    const lanes = [...grid.children].filter(
      (c) => !c.classList.contains('card') && !c.classList.contains('sel-toolbar'),
    )
    note(lanes.length === 0, `.grid has ${lanes.length} non-card child lane(s)`, lanes[0])
    const badSpans = [...grid.children].filter((c) => {
      const m = /span (\d+)/.exec(c.style.gridColumn || '')
      return m && ![4, 6, 8, 12].includes(Number(m[1]))
    })
    note(badSpans.length === 0, `${badSpans.length} card(s) with span ∉ {4,6,8,12}`, badSpans[0])
  }

  console.log(`%c✓ ${ok.length} pass`, 'color:#3ad07b', ok)
  console.log(`%c✗ ${fail.length} fail`, 'color:#ff6b6b', fail)
  return { ok, fail }
})()
```

Zero failures means the contract holds at the DOM level. Any failure points at the exact node that
drifts — paste the snippet again after each fix to confirm the regression collapses.

## Tailwind drift → typed primitive

The single biggest WC userland drift in practice: hand-rolled Tailwind class strings
(`text-12 text-text-3`, `text-13 font-semibold text-text-1`,
`text-[10px] font-semibold uppercase tracking-[0.06em] text-text-3`) where a typed `<uni-*>` element
already encodes the same contract. Tailwind tokens line up with the type scale, but free-typed
combinations re-derive what `<uni-body>` / `<uni-label>` / `<uni-eyebrow>` already bundle — and as
soon as one site uses 10 px and the next uses 11 px, the rhythm splits. Reach for the element; if
the page truly needs something off-scale, surface it as a design fork before shipping.

> Chart drift in particular: every `<uni-*-chart>` element ships as zero-dependency typed SVG that
> already matches the Dashboard chart rhythm (gridlines, padding, gradient stops, motif colours).
> Pulling in `chart.js` / `d3` re-derives the same picture with a 60 KB+ runtime, a different
> tooltip chrome, and a colour palette that drifts from `--color-brand` / `--color-status-*`. If the
> typed element truly can't express the chart, surface it as a design fork first — don't silently
> widen the dependency footprint.

The pattern: the typed `<uni-*>` element renders the **exact same DOM + class** that the Dashboard
reference uses, so userland tracks every future tweak (token shift, contrast bump, motion
preference) for free. Tailwind utilities remain the right tool for one-off layout glue (`flex`,
`gap-3`, `min-w-0`, `overflow-hidden`) — just don't re-derive typography or the canonical chrome
with them.

## Why an agent brief at all?

Documentation that works for humans assumes the reader can navigate sixteen MDX pages, follow
cross-references, and skim for the relevant section. An agent's prompt budget rewards the opposite:
**one** flat directive list, no chrome, every rule actionable from its own line. The brief above is
sized to fit comfortably inside a project-level `AGENTS.md` alongside whatever else the project
needs. The Foundations pages remain the long-form source of truth — this page is just the executive
summary an agent can keep in working memory.
