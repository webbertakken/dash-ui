# Svelte 4 → Svelte 5 migration

End state: `@w5-ui/svelte` is a modern Svelte 5 library — runes (`$props`, `$state`, `$derived`,
`$effect`), snippets (`{@render children()}`), callback event props (no `createEventDispatcher`).
`@w5-ui/wc` continues to compile to framework-agnostic custom elements via Svelte 5's updated
`customElement: true` output. Both reference apps (`dashboard-svelte`, `storybook-svelte`) compile +
test green on Svelte 5. Peer dep bumped to `svelte: ^5`. No legacy / compat shims left behind.

Scale: ~190 components, ~168 stories, ~1,300 tests, 4 consuming workspaces.

## Phase 1 — `packages/svelte` core migration

- [x] 1.1 Bump `packages/svelte/package.json`: - `peerDependencies.svelte: "^5"` -
      `devDependencies.svelte: "^5"` - Drop the "Svelte 4" wording in `description`.
- [x] 1.2 Run `sv migrate svelte-5` against `packages/svelte/src/lib/components/`. Tool converts:
      `export let` → `$props()`, `$:` → `$derived` / `$effect`, `<slot />` →
      `{@render children?.()}`, named slots → named snippets, two-way binding patterns,
      `beforeUpdate` / `afterUpdate` → `$effect.pre` / `$effect`. Commit immediately after tool run
      with `chore: sv migrate svelte-5 (auto)` so manual fixups are reviewable separately.
- [x] 1.3 Manual fixups (tool can't do all of these): - [x] `createEventDispatcher` → callback props
      (`onchange`, `onclose`, etc.). Audit list: `rg -l createEventDispatcher packages/svelte`. -
      [x] `$$slots.X` checks → snippet existence checks (`{#if footer}{@render footer()}{/if}`).
      Audit: `rg -l '\$\$slots' packages/svelte`. - [x] `$$restProps` spread → explicit `restProps`
      destructured from `$props()`. Audit: `rg -l '\$\$restProps' packages/svelte`. - [x] `on:click`
      re-forwarding inside library components → caller-passed `onclick` props. The migration tool
      flags these.
- [x] 1.4 svelte-check on `packages/svelte` (will require bumping `svelte-check` to ^4 with Svelte 5
      support — already at ^4 in the svelte package's transitive tooling, double-check). Fix every
      reported error. No suppressions.
- [x] 1.5 Update the 2 existing in-package vitest tests (`packages/svelte`) for runes-based
      component instantiation. `@testing-library/svelte` ^5 supports Svelte 5; bump if needed.
